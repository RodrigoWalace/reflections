1. Como visualizar arquivox através do comando FC (ou diff) pode ajudar a encontrar um 
bug que fôra introduzido?

O comando FC é extremamente útil para comparação de arquivos, pois com facilidade
eleexibi uma tela de compração entre os arquivos desejados revelando diretamente 
as mudanças que ocorreram entre eles, ajudando o leitor a interpretar as mudanças
ocorridas,e por meio disso, sanar a situação em que se encontrava.

2.How could having easy access to the entire history of a file make you a more
efficient programmer in the long term?

Tendo um fácil acesso a todo o histórico de um especifico arquivo, facilitará
a você entender completamente todas as modificações que foram feitas nele ao
longo do tempo, e também te ajudará a evoluir seu raciocionio faznedo com que 
você compreenda as mudanças necessárias para atingir o ponto atual do arquivo
desde o seu momento inicial.

3.What do you think are the pros and cons of manually choosing when to create a
commit, like you do in Git, vs having versions automatically saved, like Google
docs does?

Os prós são que, você escolhendo quando fazer um commit, o seu git jamais terá
informações consideradas erradas, ou inacabadas, pois você apenas as enviará
qaundo possuir certeza de suas propriedades, por outro lado, um contra é que
se outra pessoa estiver mexendo na mesma rede que você e precisar mudar a branch
em que se encontra, todas as suas alterações serão perdidas, pois você não
as salvou periodicamente, como o GoogleDocs faria, sem elvar em consideração o 
progresso atual que encontrava-se.

4.Why do you think some version control systems, like Git, allow saving multiple
files in one commit, while others, like Google Docs, treat each file separately?

Eu acho que o git permite salvarem multiplos arquivos, pois ele é uma plataforma
usada principalmente para programação, normalmente arquivos estão vinculados a 
outros, sendo assim, seria inviável enviar um arquivo sem o seu suporte, enquanto 
o googleDocs é apenas um serviço focado para armazenar documentos em uma núvem, 
como um back up, sendo assim, não existe a necessidade de upar múltiplos arquivos
em conjunto.

5.How can you use the commands git log and git diff to view the history of files?

Com o comando git log se torna possível visualizar todos os commits já realizados,
e sendo assim é fácil vizualizar uma linha do tempo das ações feitas, com o auxílio
do camando git diff é mais fácil ainda encontrar asd alterações feitas a cada passo
do código ao longo do tempo, e com isso é possível gerar um histórico completo de 
todas as modificações ja feitas.

6.How might using version control make you more confident to make changes that
could break something?

O sistema de controle de versão facilita muito o processo de confiança a medida
que sempre que uma alteração crítica está para ser feita é possível fazer um commit
sem ela estar prsente ainda, e após isso , caso algum erro indesejado surja no
decorrer do processo, basta retornar ao commit anterior para desfazer a ação e 
buscar solucionar o que ocorreu de errado.
